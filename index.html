Git

  • GitHub is a remote cloud space to save and collaborate on coding projects
  • GitHub Docs on GitHub flow was provided as a link to the workflow of modifying a repo
  • Git Commands:
    • git status: checks what branch we are currently in
    • git checkout [branch name]: navigates to an existing branch
    • git checkout -b [branch name]: creates a new branch and switches to it
    • git add: adds changes to the current branch to the staging area
    • git add -A: add ALL changes to the current branch fo the staging area
    • git commit -m "[commit message]": commits changes to current branch with message (-m) in quotes
    • git pull origin [branch]: make sure current branch is synched with base branch in GitHub. See further notes for compoents:
    • git push origin [branch]: push (modified) branch into remote GitHub branch
      • git: identify git command
      • pull: recieve branch's modifications into local environment
      • push: push modifications from local branch to remote branch
      • origin: source of the pull will be GitHub repo
      • [branch]: the branch pulled from
